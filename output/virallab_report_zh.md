# ViralLab 项目总结与应用报告

## 结论
ViralLab 已经从一个“找热点”的小工具，发展为一个覆盖“趋势发现、传播评估、内容转写、结构化改写、发布准备”的双语内容工作台。它尤其适合三类人：正在转型的专业设计师和工程师、海外留学生、以及希望把 AI 工作流引入业务的传统行业从业者和小企业主。

---

## 一、产品定位

### 1. 它是一款什么样的工具？
ViralLab 不是单纯的新闻聚合器，也不是只会写文案的 AI 工具。它更像一个面向内容决策与内容生产的双语工作台，核心目标是帮助用户完成四件事：

- 更快找到值得关注的话题
- 更清楚判断内容为什么会传播
- 更高效把视频、播客、长文转成可复用文字资产
- 更系统把原始内容整理成适合发布和复用的结构化成果

### 2. 对谁有用？
它对以下三类用户最有价值：

- 正在转型的专业设计师和工程师
- 海外留学生
- 想把 AI 工作流引入当前业务的传统行业从业者及小企业主

### 3. 我们为什么要使用它？
因为今天真正稀缺的不是“信息本身”，而是三种能力：

- 从海量信息中筛出真正有价值的内容
- 理解内容传播的底层机制，而不是只看表面热度
- 把信息快速转成结构化表达、业务动作和可复用资产

ViralLab 的价值就在于，它把这三件事放进了一条连续工作流里。

## 二、项目整体开发阶段

### 阶段一：内容发现与抓取基础
项目最早解决的是“去哪找内容”的问题。

这一阶段完成了多源抓取与内容归一化能力，包括：

- 海外内容源：DuckDuckGo News、Hacker News、Reddit、Product Hunt、YouTube Trending、英文 RSS 媒体源
- 中文内容源：36氪、虎嗅、钛媒体、爱范儿、机器之心、少数派、InfoQ 中文、人人都是产品经理等
- 基础 pipeline：统一字段、去重、排序、输出可消费结果

这一阶段的意义是，把原本分散在不同平台的信号汇总成一个可持续处理的数据入口。

### 阶段二：双语双生态产品化
第二阶段，项目开始从脚本工具演化为可长期使用的 Web 产品。

这一阶段形成了今天的站点基础结构：

- 首页统一导航
- 英文频道与中文频道分离
- Daily、News、Longform、Viral、Video to Text、Article Lab、Publish 等页面
- Render 部署能力，支持直接在线访问

这一阶段的意义是，把“开发者自己能跑”的工具，升级成“普通用户也能连续使用”的产品。

### 阶段三：传播分析与内容理解
第三阶段，项目开始补上“为什么这个内容值得关注”的判断能力。

主要能力包括：

- 使用 Jonah Berger 的 STEPPS 做传播潜力评分
- 对新闻、视频、文章和转写内容进行传播性分析
- 引入 Minto Pyramid，把文本转成“结论先行 + 要点分组 + 论据支撑”
- 将视频、音频、播客链接转换为逐字稿和 Markdown

这一阶段的意义是，项目不再只是“收集信息”，而是开始帮助用户“理解信息”。

### 阶段四：内容再生产与轻量发布闭环
目前项目已经进入工作流整合阶段，开始形成更完整的内容生产闭环。

当前已实现的方向包括：

- `Article Lab`：把 Markdown 重写成更适合传播的版本
- `Publish Queue`：内容排队、去重、质量提醒、审计记录、可选 webhook 发布
- `China Content`：国内平台内容抓取、导出、评估、外部模型改写
- 自动刷新、错误诊断、测试覆盖和 crawler 登录态支持

这意味着 ViralLab 正在从“内容辅助工具”进化为“轻量内容操作系统”。

## 三、模型的具体使用情况

这里要特别说明：ViralLab 当前并不是所有能力都重度依赖大模型。相反，它采用的是“规则化能力优先，模型能力按需插拔”的策略。

### 1. 不依赖大模型的核心能力
当前最稳定、最常用的一批能力主要来自规则引擎和本地逻辑：

- 多源内容抓取、清洗、去重、排序
- STEPPS 传播评分
- Hook、Magic Words、叙事弧线等传播信号判断
- Minto Pyramid 结构提取
- 改写建议生成
- 诊断信息分类

这类能力的优势是：

- 成本低
- 可解释
- 可重复
- 适合小团队长期使用

### 2. 可选的大模型能力
项目里已经保留了可插拔的大模型入口，主要用于更强的研究与改写场景。

当前已接入或可选的模型路线包括：

- Gemini 2.0 Flash
- OpenAI GPT-4o
- Anthropic Claude 3.5 Sonnet
- 本地 Ollama Llama 3

此外，国内内容改写页面还支持用户填写自定义 `rewrite API URL`，把内容发送给本地模型或外部改写服务。

### 3. 转写能力的实际使用方式
在转写环节，项目优先走低成本、稳定的策略：

- YouTube 优先使用现成字幕
- 没有字幕时，用 `VideoCaptioner` 做 ASR 兜底
- 对 Bilibili、Douyin、Xiaohongshu、Shipinhao、Xiaoyuzhou、播客音频等链接，走下载后转写路线
- 对 Douyin 额外提供 session 检测，帮助排查登录态和风控问题

因此，ViralLab 的“模型使用”并不是单一依赖某个大模型，而是把规则引擎、字幕源、ASR、外部模型接口整合进了一个可控流程。

## 四、价值主张：项目要解决什么问题？

### 1. 它要解决的核心问题
ViralLab 要解决的不是“没有内容”，而是以下三类更真实的问题：

- 信息很多，但很难判断什么值得看、值得做
- 知道有热点，但不知道为什么它会传播
- 有了素材之后，无法高效转成可发布、可复用、可业务化的输出

### 2. 用户能看到哪些成果？
通过 ViralLab，用户最终能看到的成果不是抽象建议，而是具体资产：

- 每日重点内容清单
- 主题化新闻与长文发现结果
- 传播评分与可解释分析
- 视频/播客/音频逐字稿
- 结构化 Markdown
- 符合传播逻辑的改写稿
- 待发布内容队列

它让“看内容”升级为“做内容、做表达、做业务决策”。

## 五、项目目前整体能实现的功能

## 海外板块
海外板块面向英文内容生态，重点帮助用户建立全球趋势视野和英文内容复用能力。

### 1. 每日内容发现
- `Daily News` 可生成每日重点内容
- 内容可按新闻、长文、播客等类型查看
- 适合用来快速掌握当天值得跟进的话题

### 2. 主题化全球搜索
- `News` 页面可按主题搜索全球热点
- 支持多源聚合，包括 Hacker News、DuckDuckGo News、Reddit、Product Hunt、NewsAPI 和英文 RSS 源
- 适合围绕 AI、设计、创业、内容营销等方向做持续跟踪

### 3. 长文与播客发现
- `Longform` 页面聚合海外高质量长文与播客
- 支持围绕职业转型、一人公司、设计与 AI 等主题聚类
- 适合做深度学习、职业判断和长期选题积累

### 4. 海外视频传播分析
- `Viral` 页面可抓取和分析 YouTube 方向的热门内容
- 对标题和描述进行传播评分
- 自动生成内容角度，帮助用户理解该题材还能怎么讲

### 5. 视频转文字与结构提炼
- `Video to Text` 支持输入视频或音频链接
- 输出逐字稿、STEPPS 分析、Minto 结构和 Markdown
- 适合把视频内容转成社媒文稿、文章、分享提纲或笔记

### 6. 文章改写与发布准备
- `Article Lab` 支持上传或粘贴 Markdown
- 自动输出更适合传播的结构化版本
- `Publish` 支持进入发布队列，做去重、质量提醒和后续 webhook 发布

### 海外板块对不同用户的实际帮助
对转型中的设计师和工程师：

- 更快建立对 AI、产品、创业、远程工作、个人品牌的全球理解
- 把英文趋势转成作品集观点、面试表达和个人内容资产

对海外留学生：

- 同时获得行业趋势、职业语言和表达素材
- 用双语视角建立比单一信息源更强的竞争力

对传统行业和小企业主：

- 学习海外成熟内容表达方式
- 借鉴全球内容案例，用于出海品牌、英文官网、英文社媒等方向

## 国内板块
国内板块面向中文内容生态，重点帮助用户理解本土平台内容机制，并快速形成适合中文平台的输出。

### 1. 中文新闻与热点发现
- 聚合 36氪、虎嗅、钛媒体、爱范儿、机器之心、少数派、人人都是产品经理、InfoQ 中文等内容源
- 帮助用户建立对中文科技、内容、商业和创作者生态的基础视野

### 2. 国内平台内容入口
- `China Content` 页面整合 B站、小红书、抖音、知乎、视频号、小宇宙、TikTok 等入口
- 提供合规优先的搜索链接与抓取路径
- 降低用户在多个平台来回切换的成本

### 3. 国内平台抓取与登录态支持
- B站支持公开接口搜索和热门内容抓取
- 小红书、抖音、知乎、视频号支持 Playwright 抓取流程
- 可保存登录态，缓解平台限制问题

### 4. 面向人群的关键词工作流
- 已内置“大厂被裁员”“海外留学生”“小老板”等受众关键词组
- 允许用户快速组织关键词，形成更贴近人群需求的内容抓取

### 5. 国内内容评估与改写准备
- 支持对国内内容做 STEPPS 评分
- 支持 Minto 结构提炼
- 支持生成改写方向
- 支持导出逐字稿 Markdown
- 支持发送到外部改写模型接口

### 6. 多平台转写与再利用
- 对 Bilibili、Douyin、Xiaohongshu、Shipinhao、Xiaoyuzhou 等内容支持转写兜底
- 把原本不易沉淀的平台内容转成可编辑、可传播、可复用的文本资产

### 国内板块对不同用户的实际帮助
对转型中的设计师和工程师：

- 学会把知识表达翻译成更适合小红书、抖音、B站的内容形态
- 把专业经验从“简历语言”升级为“公众可理解语言”

对海外留学生：

- 更快理解国内平台的表达风格和热门议题
- 在回国求职、实习、个人品牌建设上更接近真实语境

对传统行业和小企业主：

- 更快搭建本地平台内容工作流
- 用更低门槛的方式开始做内容营销、私域表达和品牌传播

## 六、针对目标群体的实际应用价值

### 1. 对专业设计师和工程师
ViralLab 的真正作用，不只是“帮你找内容”，而是帮助你完成身份升级：

- 从执行者升级为判断者
- 从被动接收信息升级为主动组织观点
- 从只会做项目升级为能把经验表达为内容资产

在 AI 时代，这种能力会直接影响求职、转型、作品集和个人品牌建设。

### 2. 对海外留学生
ViralLab 帮助留学生解决两个核心问题：

- 海外信息很多，但不容易转成自己的表达
- 中文与英文内容生态割裂，难以形成双向优势

通过 ViralLab，用户可以同时跟踪全球内容和中文平台内容，把双语背景变成认知优势和职业优势。

### 3. 对传统行业从业者与小企业主
很多传统行业用户并不缺业务经验，缺的是一条能落地的 AI 工作流。

ViralLab 的价值在于，它不是从复杂技术切入，而是从业务最容易理解的动作切入：

- 找热点
- 看案例
- 提取文字
- 改写发布

这样更适合小团队、一人团队和预算有限的真实场景。

## 七、为什么这个项目现在值得继续投入

ViralLab 有三个清晰优势：

- 它把“内容发现”和“内容生产”放在同一工作流里
- 它同时覆盖海外与国内两套生态，适合双语与跨市场用户
- 它的核心能力不完全依赖高成本模型，更容易持续运营和扩展

因此，它的产品定位可以概括为：

**一个面向职业转型、内容决策与 AI 工作流落地的双语内容操作台。**

## 八、用户引导与使用方式

### 技术型用户
技术型用户可以直接从 GitHub 下载源码并在本地运行。

1. 获取仓库  
   `git clone https://github.com/Funghi88/ViralLab.git`

2. 进入项目目录  
   `cd ViralLab`

3. 安装依赖并启动  
   `./scripts/install.sh`  
   `./scripts/dev-server.sh`

4. 浏览器访问  
   `http://127.0.0.1:5001`

如果需要更强的模型能力，可按需配置：

- `OPENAI_API_KEY`
- `GEMINI_API_KEY`
- 本地 `Ollama`
- 自定义 `rewrite API URL`

### 非技术型用户
非技术型用户可以直接访问 Render 上部署的网页版：

[https://virallab.onrender.com](https://virallab.onrender.com)

推荐使用路径：

1. 先从 `Daily News` 看当天重点内容
2. 再到 `Viral` 理解哪些内容更容易传播
3. 如需提炼素材，进入 `Video to Text`
4. 如需形成更完整输出，进入 `Article Lab`
5. 如需看中文平台和本地内容，进入 `China Content`
